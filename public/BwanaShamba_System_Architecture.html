Voice Transcript Flow

  1. User speaks → Browser SpeechRecognition API transcribes to text (shown on right side)
  2. Audio streams to Gemini Native Audio via WebSocket → AI responds with audio + thinking text
  3. AI audio plays through speaker → SpeechRecognition captures AI spoken words (shown on left side)
  4. Timing logic (isAiSpeaking + aiJustFinished) separates user vs AI transcriptions
  5. On session end → POST /api/voice-transcript saves both sides to Cloud SQL
  6. Transcript appears in Conversation History sidebar for later review

Google Cloud Services Used for Deployment

☁️ Google Cloud Run

Serverless container platform that hosts the entire application. Runs the Docker container with both Express.js and ADK services. Region: europe-west1. Scales from 0 to 3 instances.

🔨 Google Cloud Build

CI/CD service that automates the build and deployment pipeline. Builds the Docker image, pushes to Container Registry, and deploys to Cloud Run — all defined in cloudbuild.yaml.

📦 Google Container Registry (GCR)

Stores the built Docker images at gcr.io/$PROJECT_ID/bwanashamba. Cloud Build pushes images here before Cloud Run deploys them.

🗄️ Google Cloud SQL

Managed PostgreSQL 18 database instance. Connected to Cloud Run via Cloud SQL Auth Proxy. Instance: bwanashamba:europe-west1:bwanashamba-db.

🧠 Google Gemini API

AI/ML service providing Gemini 2.5 Flash for text chat, native audio for live voice, and vision capabilities for crop image analysis.

Cloud Deployment Automation Files

Yes, the application includes infrastructure-as-code files for automated deployment. All three files are committed and pushed to GitHub.

Dockerfile

Multi-stage build: Stage 1 builds the React frontend with Vite. Stage 2 creates the production image with Node.js 22, Python 3, and all dependencies. Exposes port 8080.

cloudbuild.yaml

Google Cloud Build pipeline configuration. Three steps: (1) Build Docker image, (2) Push to GCR, (3) Deploy to Cloud Run with all environment variables, Cloud SQL connection, and resource limits (1 CPU, 1Gi memory).

docker-start.sh

Startup script that runs inside the container. Starts the Python ADK Agent Service on port 8001 in the background, waits 3 seconds, then starts the Node.js Express server on port 8080.
